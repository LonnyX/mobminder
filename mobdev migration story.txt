3. secondes de délais sur la réponse d'une page de la WebApp
	
	Google indique un problème de résolution de ip 127.0.0.1 sur Win10 avec IPV6
	Afin de définir si c'est Apache ou MySQL qui a le soucis, il faut examiner le temps de réponse sur un fichier qui n'a pas d'accès DB.
	J'ai choisi /smart/check.php <= réponse immédiate. 
	
	Edit sur le fichier 
	C:\Program Files (x86)\EasyPHP-Devserver-17\eds-binaries\dbserver\mysql5717x86x201201220202\my.ini
	il faut simplement indiquer une adresse en format IPV6. L'équivalent de 127.0.0.1 est ::1
	
	bind-address = ::1
	# bind-address = 127.0.0.1
	# Solve slow access to server, check https://serverfault.com/questions/408550/connecting-to-mysql-from-php-is-extremely-slow
	# From Windows 8, IPV6 is used on Win OS and therefore you need to specify ::1 iso 127.0.0.1
